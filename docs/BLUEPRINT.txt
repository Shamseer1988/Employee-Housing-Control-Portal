PUG GROUP - ADVANCED EMPLOYEE ACCOMMODATION MANAGEMENT SYSTEM
Full Development Blueprint + Phase-by-Phase Build Prompt
Prepared for: Paris United Group / Group Companies Central Accommodation Control

====================================================================
1. SYSTEM OBJECTIVE
====================================================================

Build a centralized, multi-company, multi-branch Employee Accommodation Management Web Application for a group of companies controlling employee accommodation from Head Office.

The system must manage:
- Multiple accommodation properties in different locations
- Full buildings, partial buildings, single floors, villas, rooms, bed spaces
- Landlord master data and agreement expiry tracking
- Room and bed allocation
- Employee accommodation assignment, transfer, vacation, cancellation, and visa cancellation
- Division-wise and company-wise staff accommodation visibility
- Attachments for landlord agreements and employee documents
- Dashboards, alerts, reminders, reports, audit trails, and user privileges
- Responsive modern UI with light/dark mode and glassmorphism style

Technology direction:
- Backend: Python Flask
- Database: PostgreSQL
- ORM/Migration: SQLAlchemy + Alembic / Flask-Migrate
- Frontend: Next.js + TypeScript + React
- Styling: Tailwind CSS, shadcn/ui, Framer Motion, GSAP where needed
- API: REST API with possible future WebSocket notification support
- Authentication: JWT/session-based secure login
- Authorization: Role Based Access Control and optional permission matrix
- Deployment: Ubuntu server, Gunicorn/Uvicorn as applicable, Nginx reverse proxy, Cloudflare DNS/SSL

4. TRANSACTION MODULE
====================================================================

4.1 Employee Room Assignment

Purpose:
Assign an employee to a property, room, and bed.

Fields:
- Transaction Number
- Employee
- Division
- Property
- Floor
- Room
- Bed
- Assignment Date
- Expected Stay Period
- Reason
- Approved By
- Attachment if any
- Remarks

Rules:
- Only empty beds can be assigned
- Employee cannot have two active bed allocations
- Bed status changes to Occupied after posting
- Employee master updates current property/room/bed
- Transaction history created

4.2 Employee Room Transfer

Purpose:
Move employee from one room/bed/property to another.

Fields:
- Employee
- Current Property/Room/Bed
- New Property/Room/Bed
- Transfer Date
- Reason
- Approved By
- Remarks

Rules:
- Old bed becomes Empty
- New bed becomes Occupied
- Full history retained
- Transfer requires permission or approval

4.3 Bed Change

Purpose:
Change bed inside same room.

Rules:
- Old bed becomes Empty
- New bed becomes Occupied
- Room remains same
- History retained

4.4 Room Change

Purpose:
Change room within same property or different property.

Rules:
- Similar to transfer
- Must validate available bed
- Optional approval workflow

4.5 Room/Bed Cancellation

Purpose:
Cancel employee accommodation allocation.

Reasons:
- Resigned
- Terminated
- Visa Cancelled
- Shifted outside company accommodation
- Vacation
- Other

Rules:
- Bed becomes Empty
- Employee accommodation status becomes Not Assigned
- Cancellation reason mandatory
- Cannot delete history

4.6 Employee Vacation

Purpose:
Track employees going on leave while maintaining or releasing accommodation.

Fields:
- Employee
- Vacation Start Date
- Vacation End Date
- Keep Bed Reserved: Yes/No
- Property/Room/Bed
- Remarks

Rules:
- If bed reserved, status remains Reserved
- If not reserved, bed becomes Empty
- Dashboard must show employees on vacation

4.7 Visa Cancellation / Final Exit

Purpose:
Release accommodation due to visa cancellation or final exit.

Rules:
- Accommodation must be cancelled
- Bed becomes Empty
- Employee status becomes Visa Cancelled / Final Exit
- Attachment can be uploaded
- History retained

4.8 Landlord Agreement Renewal

Purpose:
Renew property agreement and update contract details.

Fields:
- Property
- Landlord
- Old Agreement Expiry
- New Agreement Start Date
- New Agreement Expiry Date
- New Monthly Rent
- Renewal Attachment
- Remarks

Rules:
- Previous agreement archived
- New expiry reminder generated
- Dashboard updates

4.9 Property Maintenance / Room Maintenance

Purpose:
Block property, room, or bed for maintenance.

Rules:
- Occupied room cannot be blocked unless employee is transferred
- Maintenance status visible in dashboard
- Maintenance completion updates status

4.10 Bulk Allocation / Bulk Transfer

Purpose:
Upload Excel to assign or transfer multiple employees.

Rules:
- Template download required
- Validate all rows before posting
- Show error sheet for failed rows
- No partial posting unless user confirms

- All transactions must have created_by, created_at, updated_by, updated_at

- Attachments should be linked generically using entity_type and entity_id
- Use indexes on employee_id, property_id, room_id, bed_id, division_id, status, expiry dates
- Use PostgreSQL constraints to prevent duplicate active bed allocation
